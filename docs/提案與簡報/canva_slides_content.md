### 對比表

| 比較項目 | 一般 LLM | 小勞鼠（Hybrid RAG） |
|---------|---------|-------------------|
| **資料來源** | 模型記憶，可能過時 | 官方 PDF + 裁罰 CSV + GraphRAG 即時推理 |
| **推理方式** | 文字問答，無邏輯鏈 | 事故 → 法規 → 義務 → 文件 → 罰則（有推理鏈） |
| **可追溯性** | 難確認引用依據 | 可回到圖譜節點、法條原文、裁罰案例 |
| **產業專注** | 泛用，不懂 IT 情境 | 專注資訊服務業 8 種工作情境 |
| **輸出格式** | 建議文字 | 可執行 SOP / 文件清單 / 風險清單 |
| **案例對標** | 無 | 58,496 筆裁罰資料，同業違規精確對標 |

### 底部備註
```
一般 LLM 知道法律，小勞鼠知道你的公司會被怎麼罰
```

### 右側：AI 輸出重點（條列）

**職災判定**
- ✅ S004（駐點機房）→ A003（搬運受傷）→ 職災成立初判

**適用法條**
- 勞基法§59：工資補償、醫療補償
- 勞基法§63-1：原事業單位連帶責任

**企業義務**
- D002 工資補償（平均工資全額）
- D003 醫療補償（全額）
- 8 小時內通報職安署重大職災

**必備文件**
- 診斷證明書 / 醫療收據

## 第 13 頁｜Demo 2：精準止損

**① 資遣理由確認**

**② 預告期（依年資）**

| 年資 | 預告天數 |
|------|---------|

| 1 年以上至 3 年未滿 | 20 天 |

**⑤ 法律風險**

### 雙面輸出對比表

| 項目 | 🏢 HR 端（企業視角）| 👷 勞工端（員工視角）|
|-----|-------------------|-------------------|
| **判定** | 確認通勤事故 / 公出事故 / 職災成立 | 可申請職業傷病給付、醫療給付 |
| **義務 / 權益** | 公司補償義務、重大職災通報義務 | 失能給付、照護補助、職災醫療 |
| **必備文件** | 派工單、出差申請、行程紀錄 | 診斷證明、事故證明、薪資資料 |
| **申請期限** | 8 小時內通報（若死亡） | 傷病給付：2 年內申請 |
| **受理單位** | 職安署（通報）、勞動局（查核）| 勞保局（給付）、勞工局（調解）|
| **後續風險** | 業務過失傷害刑責、民事賠償 | 調解路徑、申訴機制 |

### 底部標語
