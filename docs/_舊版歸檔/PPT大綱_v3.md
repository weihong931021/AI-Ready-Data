# PPT 大綱 v3：資訊服務業事故處置與合規風險作戰室

> 15 頁初賽提案建議稿。主軸是資訊服務業 / 系統整合業的駐點客戶現場、IDC 維運、通勤公出與職場不法侵害風險；上線加班與過勞保留為輔助查詢情境。

## 1. 封面

- 題目：資訊服務業事故處置與合規風險作戰室
- 副標：Hybrid RAG 驅動的駐點責任、客戶現場事故與職場不法侵害風險推理平台
- 放隊名、組員、slogan。

## 2. 痛點：資訊服務業風險不是查一條法規就能解

- 駐點工程師在客戶現場受傷時，HR、PM、法務要釐清責任邊界。
- IDC / 機房維運、客戶拜訪與公出途中事故，需要快速確認證據、責任與處置流程。
- 客戶、主管或同事不法侵害 / 性騷擾若處理失當，會變成裁罰、訴訟與聲譽風險。

## 3. 解決方案：資訊服務業風險作戰室

- 企業端回答「這是客戶現場事故、IDC 維運事故、通勤公出，還是不法侵害」。
- 系統輸出證據缺口、HR/PM/法務/職安分工、給付補償、裁罰與治理風險。
- 勞工端保留為第二視角，回答「我能申請什麼、要準備什麼、如何申訴」。

## 4. 資料來源總覽

- VectorRAG：32 份 PDF，新增過勞與職場不法侵害官方指引。
- GraphRAG：17 種節點、20 份關係、2,303 條邊。
- Risk Layer：26,513 筆完整裁罰資料層。

## 5. Gemini Data 平台操作：資料清理與匯入

- 法規 PDF 分批上傳到 Documents。
- CSV 分成節點、關係、裁罰補充三類。
- 裁罰資料統一日期、金額、來源、產業與案例 ID。

## 6. Gemini Data 平台操作：知識圖繪製

- 顯示事故、法規、義務、權益、文件、案例、刑責、民賠、治理風險節點。
- 圖上標示主要推理鏈。

## 7. Gemini Data 平台操作：設定節點與關係

- 展示 Category + Unique Key。
- 節點先上，關係後上。
- R9/R12/R13 拆成單一起點 Category，避免 Data Flow 斷邊。

## 8. Hybrid RAG 架構

- VectorRAG：理解口語事件並引用官方法規、過勞指引與不法侵害指引。
- GraphRAG：工作情境 → 事故/法規 → 證據 → 任務 → 角色分工。
- Risk Layer：資訊服務業同業裁罰、法條熱點、月份趨勢、產業風險。

## 9. Robot Setting / Context Engineering

- 身分優先：人資、勞工、工會。
- 回答需分成認定、義務/權益、文件、期限、風險、下一步。
- 法規不確定時標示需人工確認。

## 10. Demo 1：駐點工程師客戶機房受傷

- 工程師到客戶機房搬伺服器或維修設備時受傷。
- 展示派工、門禁、機房作業申請、合約範圍與診斷證明如何釐清責任。
- 輸出 HR、PM、法務、職安與客戶窗口的處置任務。

## 11. Demo 2：客戶或主管職場不法侵害 / 性騷擾

- 專案現場主管長期辱罵工程師，或客戶窗口對駐點人員性騷擾。
- 展示申訴、調查、保護措施、諮商轉介與避免報復任務。
- 引用職安署不法侵害預防指引與性平法義務。

## 12. Demo 3：通勤 / 公出事故雙面介面 + 同業裁罰風險

- 工程師去客戶端途中車禍，展示人資端與勞工端的不同輸出。
- 查資訊服務業常見災保法違規、投保薪資、未投保與紀錄缺口。
- 長工時/過勞問題保留為 Q&A 輔助查詢，不放在主 demo 開場。

## 13. 介面初步構想

- 身分切換。
- 對話輸入。
- 右側即時推理路徑。
- 人資端期限倒數與風險卡。

## 14. 商業價值

- 中小企業：降低法遵查詢與顧問成本。
- 人資顧問/律所：前期諮詢工具。
- 工會/法扶：快速整理勞工權益。
- 地方勞工局：自助查詢分流。

## 15. 結論

- 這不是單純法規查詢，而是企業合規風險推理。
- 必須結合 Gemini Data 的 VectorRAG + GraphRAG 才能同時處理原文與關係。
- 後續可串 EAP API 做前端 Agent。
