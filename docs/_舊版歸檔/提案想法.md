# 題目：資訊服務業事故處置與合規風險管理助手（含勞工權益雙面介面）

> 原題目方向「勞工補償、請領與申訴路徑智慧助手」轉向後的最終提案。
> 同一套知識圖譜，雙面介面：人資端看「企業義務」，勞工端看「個人權益」。

---

## v3 定位更新

本提案目前定位為 **資訊服務業事故處置與合規風險作戰室**。主產品面向企業人資、法務、PM 與管理層，優先回答「駐點、客戶現場、IDC、通勤公出或職場不法侵害發生後，公司該做什麼、需要哪些證據、誰要負責、會有哪些裁罰與治理風險」；勞工端保留為同一知識庫的第二視角，用來展示整合創新與社會價值。

核心資料不只法條查詢，而是三層整合：

- **VectorRAG**：32 份 PDF，包含官方法規、勞保局給付指南、重大職災統計、案例彙編，以及職場不法侵害與異常工作負荷等官方指引。
- **GraphRAG**：17 種節點、20 份關係，串起工作情境、事故、法規、證據、處置任務、企業角色、義務、權益、刑責、民賠與治理風險。主線先處理責任邊界清楚的事故與客戶場域情境，長工時/過勞保留為查詢與輔助判斷能力。
- **Risk Layer**：26,513 筆完整裁罰資料層，支援同業罰鍰、法條熱點、月份趨勢與產業風險。

## 一、痛點與市場機會

當駐點、客戶現場、IDC、通勤公出或職場不法侵害事件發生，**企業人資 / 法務 / PM**必須在極短時間內判斷：

- 這是公司內部事故、客戶現場事故、通勤公出，還是不法侵害？
- 需要哪些證據釐清責任邊界，例如派工、門禁、合約、診斷、申訴或調查紀錄？
- HR、PM、法務、EHS 與客戶窗口各自要做什麼？
- 後續如何避免升級為裁罰、勞資爭議、訴訟或客戶合約爭議？

相關資訊散落於《勞基法》、《勞工職業災害保險及保護法》、《職業安全衛生法》、性別平等工作法、勞保局給付手冊、職安署指引與裁罰公告。企業真正困難的不是查單一法條，而是把事故情境、證據、角色分工與後續責任串成可執行流程；一旦處理失當，**罰鍰、補償、訴訟、客戶爭議與聲譽成本都會擴大**。

對**勞工**而言則是反向問題：不知道自己有哪些權益、能申請哪些給付、應準備哪些文件，常因錯失 5 年申請時效而失權。

→ 一套知識庫、兩個視角、共用同一份法規邏輯。

---

## 二、解決方案：企業風險主軸 × 勞工權益雙面介面

```text
                ┌──────────────────────────┐
                │   職災法規知識圖譜 (共用)  │
                │   VectorRAG + GraphRAG    │
                └──────────┬───────────────┘
                           │
              ┌────────────┴─────────────┐
              ▼                          ▼
      ┌───────────────┐          ┌───────────────┐
      │  企業人資介面  │          │ 勞工 / 工會介面 │
      │  「我該做什麼」│          │  「我能要什麼」  │
      └───────────────┘          └───────────────┘
```

進入系統先選擇身份 → 輸入**同一個事故情境** → 系統依身份輸出企業義務/風險或勞工權益。企業端會額外顯示同業裁罰、民事賠償、刑事責任與治理風險。

**為何雙面設計？**

1. 資料來源完全相同，零重複建置成本
2. 完整對應評分中的「**整合創新**」（同一 GraphRAG 兩種推理路徑）
3. Demo 視覺效果強：左右並列展示同一事故對人資 vs 勞工的不同意義

## 三、Hybrid RAG 架構

### VectorRAG（語意檢索層）

**處理對象**：使用者口語化、情緒化的事故描述

- 範例輸入：「我同事昨天騎車去送貨被撞，現在住院」
- 檢索素材：法條原文、勞動部函釋、FAQ、給付手冊、匿名化職災案例
- 任務：把白話事故映射到正式法律分類（→「通勤職災」）

### GraphRAG（關聯推理層）

**節點（Node）類型**：

- 事故類型（職業病 / 工傷 / 通勤事故）
- 適用法規（條文層級）
- 企業法定義務
- 勞工可請領給付
- 申報 / 申請期限
- 所需文件
- 罰則
- 受理單位（勞保局 / 地方勞工局 / 職安署）
- 申訴 / 調解程序
- 真實裁罰案例與產業統計
- 雇主刑事責任、民事賠償、公司治理風險

**關係（Edge）類型**：
適用於 / 衍生義務 / 衍生權益 / 需文件 / 對應事故 / 違反法條 / 衍生刑責 / 衍生民賠 / 影響治理 / 負責人連帶

**雙路徑推理範例**（同一節點起點，不同方向遍歷）：

```text
                 [通勤事故]
                     │
              [災保法§X 條]
                     │
       ┌─────────────┴─────────────┐
       ▼                           ▼
  人資路徑                      勞工路徑
  → 8 小時通報                  → 傷病給付
  → 應備事故報告書              → 申請條件 + 5 年時效
  → 違規罰鍰 3–30 萬            → 申請文件清單
  → 後續調解流程                → 受理單位
```

## 四、AI Agent + Vibe Coding 介面設計

1. **身份選擇頁**：人資 / 勞工 / 工會幹部
2. **事故情境輸入**（自然語言對話）
   - 例：「員工昨天在工地搬物時腰部受傷送醫」
3. **AI Agent 推理流程可視化**
   - VectorRAG 抓相關條文與類似案例 → 顯示信心分數
   - GraphRAG 動態展開義務 / 權益鏈 → 動畫呈現節點串接
   - 最終輸出：**待辦清單 + 期限倒數計時 + 文件範本下載**
4. **EAP 專案 API 串接**（加分項）
   - 模擬串接企業 OA / HR 系統，自動建立通報任務與簽核流程
   - 對勞工端：模擬串接勞保局 e 化系統查詢申請進度

## 五、商業 / 社會價值

| 對象 | 價值 |
| --- | --- |
| **中小企業（50–300 人）** | 取代每次 3–5 萬的合規諮詢費，月費級訂閱制 |
| **目標產業** | 製造、營造、物流、餐飲（人資專業度有限但職災高風險） |
| **工會 / 法扶單位** | 即時查詢勞工權益，協助會員主張時效 |
| **地方勞工局** | 自助查詢分流，減少臨櫃諮詢負荷 |
| **律師事務所** | 接案前期客戶諮詢工具 |
