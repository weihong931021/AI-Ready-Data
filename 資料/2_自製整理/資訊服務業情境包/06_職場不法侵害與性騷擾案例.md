# 職場不法侵害 / 性騷擾 / 霸凌案例

## 情境摘要

專案團隊長期面臨高壓管理，主管以辱罵、羞辱、威脅績效等方式對待工程師；或工程師在客戶端駐點期間遭客戶人員性騷擾、言語攻擊或不當對待，公司未即時處理。員工後續出現身心症狀、請假或離職爭議。

這類案例不是傳統職災主線，但非常適合 B2B 提案，因為它連到 ESG、DEI、職場安全、客戶管理與雇主防治義務。

## RAG 系統要解決的問題

- 事件是否屬於執行職務遭受不法侵害或性騷擾？
- 公司是否已有申訴、調查、保護與改善流程？
- 若行為人是客戶端人員，公司仍應採取哪些措施？
- 需要哪些紀錄支撐申訴處理與後續改善？
- 如何避免個案演變成勞資爭議、裁罰或商譽風險？

## 需要蒐集的證據

| 證據 | 用途 |
| --- | --- |
| 申訴紀錄 | 確認事件內容、時間、對象與訴求 |
| 調查訪談紀錄 | 支撐公司是否啟動處理程序 |
| 專案會議與訊息紀錄 | 釐清不當言行、指揮關係與發生場景 |
| 諮商 / 醫療 / 工作調整紀錄 | 證明保護措施與員工身心影響 |
| 客戶派工與駐點範圍 | 判斷客戶現場責任與公司協調義務 |
| 教育訓練與預防措施紀錄 | 判斷公司制度是否完整 |

## 對應法規與指引

- 職業安全衛生設施規則第 324-3 條：執行職務遭受不法侵害預防措施
- 性別平等工作法第 13 條：雇主防治性騷擾義務
- 性別平等工作法第 38-1 條：違反防治義務罰鍰
- 民法第 483-1 條：僱用人安全保護義務
- 執行職務遭受不法侵害預防指引

## Demo 問句

```text
工程師在客戶端駐點期間遭客戶主管長期言語羞辱，公司收到申訴後應該啟動哪些流程？要保留哪些紀錄？
```

## 預期回答重點

1. 先判斷是否屬於執行職務遭受不法侵害或性騷擾相關風險。
2. 要求建立申訴紀錄、訪談紀錄、訊息紀錄、保護措施與工作調整紀錄。
3. HR 要啟動申訴處理與保護措施，法務評估客戶責任與合約通知，PM 協調駐點安排。
4. 若涉及性騷擾，提醒雇主防治、調查、處置義務與罰鍰風險。
5. 商業面指出客戶場域不等於公司可以放手，駐點人員仍需要明確保護機制。

## 商業價值

此案例能強化提案的現代企業管理價值，讓系統不只是職災工具，也能處理職場安全、DEI 與客戶治理問題。
