# 駐點工程師客戶現場受傷案例

## 情境摘要

工程師依公司派工到客戶機房維修設備。作業過程中搬運伺服器、整理線路或進出機房時跌倒受傷。事故地點在客戶場域，但人員受僱於資訊服務公司，且可能涉及客戶現場管理、承攬契約、派工紀錄與工作指揮權。

這個案例的價值在於展示 B2B 責任邊界：事故不是發生在自己公司辦公室，但企業仍可能需要處理職災認定、補償義務、客戶協作與合約責任。

## RAG 系統要解決的問題

- 這是通勤、公出、職務執行，還是客戶場所事故？
- 公司、客戶、承攬或派遣關係如何影響責任判斷？
- 要蒐集哪些資料才能還原事故地點、作業內容與指揮鏈？
- 是否需要進行職災補償、勞保給付或民事責任評估？
- PM 與法務是否要檢查合約中的客戶現場安全責任？

## 需要蒐集的證據

| 證據 | 用途 |
| --- | --- |
| 客戶派工單 | 證明受公司指派到客戶現場工作 |
| 門禁紀錄 | 證明進出客戶場域時間 |
| 機房作業申請 | 證明作業內容、區域與客戶核准紀錄 |
| incident ticket 或維修紀錄 | 證明作業原因與技術處理內容 |
| 合約 / 駐點服務範圍 | 判斷責任邊界與工作指揮權 |
| 診斷證明 | 連結受傷結果與補償 / 給付流程 |
| 現場照片與事故紀錄 | 支撐事故原因與安全管理缺口 |

## 對應法規與指引

- 勞動基準法第 59 條：職業災害補償
- 民法第 483-1 條：僱用人保護受僱人安全義務
- 民法第 188 條：僱用人責任
- 職業安全衛生法第 6 條：危害預防措施
- 職業安全衛生法第 37 條：重大職災通報
- 勞工職業災害保險及保護法：傷病、失能、死亡、醫療給付

## Demo 問句

```text
工程師到客戶機房維修設備時搬伺服器受傷，事故地點在客戶端。公司要先確認哪些責任邊界與證據？
```

## 預期回答重點

1. 先確認是否為受公司指派執行職務。
2. 要求蒐集派工單、門禁、作業申請、維修紀錄、合約與診斷證明。
3. 區分公司對員工的補償義務、客戶現場安全協作責任、合約責任。
4. 若屬重大職災，提醒通報與現場保留。
5. 提醒 PM 與法務回頭檢查駐點服務範圍、客戶現場安全條款與事故通報 SLA。

## 商業價值

這個案例能呈現系統整合商最常見的責任灰區，比單純工地事故更貼近 B2B 商業場景。
