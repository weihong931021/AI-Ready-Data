# IDC / 機房維運事故案例

## 情境摘要

維運人員在 IDC 或企業機房處理機櫃、電源、線路、備援設備或緊急維修時，發生觸電、跌倒、搬重物拉傷、被設備壓傷等事故。此類事故不像營造業高空墜落，但仍屬資訊服務業真實存在的職安風險。

## RAG 系統要解決的問題

- 事故是否屬於職務執行中的職災？
- 是否涉及機房安全、電氣危害、搬運作業或夜間維運疲勞？
- 是否需要通報、保留現場或進行事故調查？
- 哪些紀錄能證明事故發生於公司或客戶要求的維運工作？
- 公司是否有機房作業 SOP、教育訓練與危害預防措施？

## 需要蒐集的證據

| 證據 | 用途 |
| --- | --- |
| 機房作業申請 | 證明作業項目與核准時間 |
| 門禁紀錄 | 證明進出機房時間 |
| incident ticket | 證明維運任務來源與緊急程度 |
| 系統上線 / 維護窗口紀錄 | 判斷是否夜間、假日或高壓作業 |
| 現場照片與事故紀錄 | 釐清危害來源 |
| 教育訓練與 SOP 紀錄 | 判斷公司是否盡預防義務 |
| 診斷證明 | 連結傷害與給付 / 補償流程 |

## 對應法規與指引

- 職業安全衛生法第 6 條：雇主危害預防義務
- 職業安全衛生法第 37 條：重大職災通報
- 民法第 483-1 條：僱用人安全保護義務
- 勞動基準法第 59 條：職災補償
- 勞工職業災害保險及保護法：醫療、傷病、失能、死亡給付

## Demo 問句

```text
IDC 維運人員半夜進機房處理電源異常時受傷，公司要怎麼判斷是否涉及職災與職安義務？
```

## 預期回答重點

1. 判斷事故是否發生於公司指派或客戶要求的維運任務。
2. 蒐集作業申請、門禁、ticket、維護窗口、現場照片、診斷證明。
3. 檢查職安法危害預防、重大職災通報、現場保留與調查協助。
4. 若涉及夜間或長時間維運，也要接到異常工作負荷風險。
5. 建議企業盤點機房作業 SOP、教育訓練、雙人作業與緊急應變制度。

## 商業價值

此案例可讓評審看到：資訊服務業也有具體職安風險，不只是辦公室白領風險。
