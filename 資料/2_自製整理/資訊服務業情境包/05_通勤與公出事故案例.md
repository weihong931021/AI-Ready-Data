# 通勤與公出事故案例

## 情境摘要

工程師、顧問、業務或 PM 在前往客戶端開會、維修、教育訓練、系統驗收途中發生車禍或跌倒受傷。若路線、時間與工作任務具有合理關聯，可能涉及通勤災害或公出事故。

## RAG 系統要解決的問題

- 事故是一般通勤、因公外出，還是私人行程？
- 是否有派工、會議、客戶拜訪或公司指派紀錄？
- 可以申請哪些保險給付或補償？
- HR 要求員工補哪些資料才足以初判？
- 公司是否需要改善交通安全、外勤管理或公出保險制度？

## 需要蒐集的證據

| 證據 | 用途 |
| --- | --- |
| 客戶派工單或拜訪紀錄 | 證明外出目的與公司指派 |
| 會議邀請 / 行事曆 | 證明時間與地點 |
| 交通紀錄 | 支撐移動路線與合理性 |
| 門禁或簽到紀錄 | 證明到達客戶端或公司場域 |
| 診斷證明 | 連結事故傷害與給付流程 |
| 警方事故紀錄 | 支撐事故發生事實 |

## 對應法規與指引

- 勞工職業災害保險及保護法第 27 條：通勤災害保險給付準用
- 勞工職業災害保險及保護法第 42 條：傷病給付
- 勞工職業災害保險及保護法第 43 條：失能給付
- 勞動基準法第 59 條：職業災害補償
- 民法第 188 條：僱用人責任，視具體個案判斷

## Demo 問句

```text
工程師去客戶端處理系統問題途中發生車禍，HR 要如何判斷需要補哪些資料，以及可能有哪些給付？
```

## 預期回答重點

1. 先釐清是否為公司指派、合理路線、合理時間與工作目的。
2. 蒐集派工單、會議紀錄、交通紀錄、診斷證明、事故紀錄。
3. 提醒可能涉及通勤災害或公出事故。
4. 若符合條件，連到醫療、傷病、失能或死亡給付，以及雇主補償義務。
5. 建議公司建立外勤與客戶拜訪紀錄，避免事後無法證明工作關聯。

## 商業價值

此案例能覆蓋白領、顧問、PM、業務、工程師等高頻工作模式，適合展示系統的普適性。
