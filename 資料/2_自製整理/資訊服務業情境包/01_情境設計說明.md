# 資訊服務業情境包設計說明

## 目的

本情境包用來補強「資訊服務業事故處置與合規風險」主軸，讓系統不只回答法條，而是能把工作情境、證據資料、企業角色、處置任務與裁罰風險串成可操作的判斷流程。主 demo 先聚焦在事故、客戶場域責任與不法侵害；長工時、加班與過勞則保留為平台可查詢的輔助情境。

適用對象包含系統整合商、資訊服務公司、軟體開發團隊、維運中心、IDC / 機房維運團隊，以及常態駐點客戶現場的專案人員。

## 為什麼要自製整理

官方法規與指引通常回答「規定是什麼」，但企業實務更常問：

- 這件事算不算可能職災？
- 我現在要蒐集哪些證據？
- HR、PM、法務、職安或主管各自要做什麼？
- 這是公司內部責任、客戶現場責任，還是派遣 / 承攬邊界問題？
- 如果沒有處理，可能引發哪些裁罰、補償、民事或公司治理風險？

因此本包把資訊服務業高頻情境整理成 demo 可用的中間層資料，補足 GraphRAG 與 VectorRAG 之間的落差。

## 五層設計

| 層級 | 內容 | 對應檔案或資料 |
| --- | --- | --- |
| 情境層 | 駐點、IDC、客戶現場、通勤公出、不法侵害；系統上線與 on-call 作為輔助查詢 | `14_工作情境.csv`、本包案例文件 |
| 證據層 | 工時、值班、ticket、門禁、派工、診斷、申訴、調查 | `15_證據資料.csv`、`07_證據資料欄位樣本.csv` |
| 流程層 | HR、PM、法務、EHS、主管、員工、客戶窗口各自任務 | `16_處置任務.csv`、`17_企業角色.csv`、`08_處置流程樣本.csv` |
| 判斷層 | 是否可能職災、是否有異常工作負荷、是否要通報、是否補償或給付 | `R15`、`R16`、`R17`、`R19` |
| 商業層 | 裁罰風險、內控缺口、客戶合約責任、公司治理風險 | 完整裁罰資料層與 demo 問答 |

## 主 demo 建議

1. 駐點工程師客戶現場受傷
2. 客戶或主管職場不法侵害 / 性騷擾
3. 通勤與公出事故
4. IDC 機房維運事故
5. 資訊服務業同業裁罰風險雷達

系統上線長工時 / 過勞風險保留為輔助查詢情境：平台可以回答工時、加班、on-call、異常工作負荷與職災認定問題，但不把它作為第一個商業主 demo。

工地墜落案例仍可保留作為高風險極端案例，但不應該定義產品主軸。

## Demo 定位句

面向資訊服務與系統整合企業的事故處置與合規風險作戰室，協助 HR、法務、專案管理者與職安人員處理駐點客戶現場、IDC 維運、通勤出差、職場不法侵害與職災補償等高頻風險；長工時與過勞則作為可查詢、可補充判斷的風險題庫。
