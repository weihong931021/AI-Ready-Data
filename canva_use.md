## 第 2 頁｜解決方案：一個知識庫，兩種決策視角

版型：左右對稱，中間放 Hybrid RAG。

**企業端回答：**
- 公司現在該做什麼
- 誰負責
- 少做會怎樣
- 需要哪些文件
- 有沒有刑責 / 罰鍰 / 民事賠償

**勞工端回答：**
- 我能申請什麼
- 要準備什麼文件
- 申請期限
- 受理單位
- 後續申訴或調解路徑

## 第 3 頁｜應用與使用情境

| 情境 | 觸發時機 | 使用者 | AI 輸出 |
| --- | --- | --- | --- |
| **應急導航** | 職災、客戶現場、IDC、通勤公出、不法侵害 | HR / PM / EHS | 事故判定、SOP、文件、罰則、刑責 |
| **精準止損** | 資遣 / 解僱前 | HR / Legal | 程序、期限、舉證、調解風險 |

**底部三句：**
> 事故發生時，每小時都是罰款計時器
> 資遣不合規，調解輸了是天價
> 同業被罰什麼，就是你的下一個風險

## 第 4 頁｜傳統 LLM vs JobShield

| 比較項目 | 一般 LLM | JobShield |
| --- | --- | --- |
| 資料來源 | 可能憑模型記憶 | 官方 PDF + 裁罰 CSV + GraphRAG |
| 推理方式 | 文字回答 | 事故 → 法規 → 義務 → 文件 → 罰則 |
| 可追溯性 | 難確認依據 | 可回到節點、法條、案例 |
| 產業性 | 泛用 | 專注資訊服務業 |
| 輸出 | 建議 | 可執行 SOP / 風險清單 |

```
工作情境 S
  → 事故 A
  → 法規 L

```

  → D002 工資補償 / D003 醫療補償
  → DOC001 診斷證明 / DOC003 工資清冊
  → CR002 業務過失傷害

## 第 13 頁｜Demo 2：精準止損

**問句：**
> 我要資遣表現不佳的工程師，有哪些程序？期限是什麼？不照做會怎樣？

**重點：**
- 確認資遣理由（需書面績效紀錄）
- 預告期（年資 4 年以上需 30 日）
- 資遣費計算
- 非自願離職證明書
- 工資 / 出勤 / 績效紀錄保全
- 勞動事件法§37 工資推定（雇主不利條款）
- 大量解僱需 60 日前提出計畫書
