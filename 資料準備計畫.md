## 一、平台規範速查（必讀）

### 1.1 檔案格式

| 格式 | 用途 | 對應 RAG |
| --- | --- | --- |
| **PDF** | 法條、函釋、FAQ、給付指南、案例 | VectorRAG |
| **xlsx** | 結構化節點與關係表 | GraphRAG |

### 1.2 xlsx 嚴格規範

- **第一列必須為欄位名稱**
- **禁止合併儲存格**
- 一格只放一個值，**禁止陣列**（如 `DOC001, DOC002` 違規）
- 不同主題的表 **拆成多個檔或多個 sheet**
- 日期欄位統一 `YYYY-MM-DD`
- 數值欄位純數字（不要寫「30 萬」要寫 `300000`）

### 1.3 圖譜核心機制

- **Data Source（原始檔）+ Data Model（藍圖）→ Data Flow**，按 Start 才會寫入畫布
- 系統用 **Category + Unique Key** 判斷節點是否重複
  - 同 Category + 同 Unique Key → 自動合併為同一節點
  - 不同 Category 即使 Unique Key 相同也視為不同節點
- 同一 Source 可搭配不同 Model（一對多）；不同 Source 可套同一 Model（多對一）
- **編輯 Data Flow 後務必「清空畫布」再全部重跑**

### 1.4 平台網址 / API

- 平台：`https://cloud.geminidata.com`（Chrome）
- API 文件：`https://cloud.geminidata.com/api/docs/`
- 核心 endpoint：`POST /api/v1/chat/create` → `POST /api/v1/chat/{chat_id}` → `GET /api/v1/chat/{chat_id}/messages`

## 四、Robot Setting / 系統提示詞（範例）

依工作坊範例改寫，初賽 demo 可直接用：

```text
你是「職災法規智慧助手」，依照使用者身分（人資 / 勞工）回答事故對應的法定義務或權益。

回覆規則：
1. 使用繁體中文，台灣用語。
2. 回答前先確認身分（人資 / 勞工）。
3. 若身分為「人資」：列出法定義務、申報期限、所需文件、違反罰則、處分單位。
4. 若身分為「勞工」：列出可申請給付、申請條件、時效年限、所需文件、受理單位。
5. 引用條文時請標明法規名稱與條號。
6. 若資訊不足以判定為職災，回應「需進一步釐清以下資訊：...」並列出待確認項目。
7. 每次回答結尾生成 3 個延伸問題範例。

Generate Cypher 時使用 Neo4j v5 語法，不要使用 OVER()。
```
