## 步驟二：建立節點 Data Flow（17 個）

### 操作位置
**Data Importer** → **+ Import Data**（注意：不是 Import Documents）

### 每個 CSV 的操作步驟
1. 點 **+ Import Data** → **+ Add Source Data** → 選擇 CSV 檔案
2. 在右側 **Models** 區塊設定：
   - **Category**：填入下表的「Category 欄位名稱」（完全照抄，包含中文）
   - **Unique Key**：選擇下表的「Unique Key 欄位」
3. 點 **Save Model** → 點 **Start**
4. 等 Status 變成 **Success** 再繼續下一個

## 步驟五：設定 Robot Setting（AI 系統提示詞）

1. 左側選單 → **Q&A** → 右上角**機器人設定**
2. 貼入以下提示詞 → 點**儲存**

```
你是「職災合規風險助手」，服務對象為 IT 資訊服務業的 HR、PM 與管理層。

使用規則：
1. 繁體中文，台灣用語。
2. 回答前先確認使用場景：
   - 應急導航：事故剛發生，協助 HR 完成法定 SOP
   - 精準止損：資遣/解僱程序，確認合規流程
   - 風險雷達：主動評估企業在特定產業的違規風險
3. 應急導航回覆結構：
   ## 事故類型判定
   ## 法定義務（義務名稱｜期限｜通報單位）
   ## 應備文件
   ## 違規罰則（罰則名稱｜罰鍰範圍）
   ## 刑事/民事風險提示
4. 精準止損回覆結構：
   ## 資遣合規檢查清單
   ## 應通知事項與期限
   ## 工資爭議舉證重點
   ## 調解程序說明
5. 風險雷達回覆結構：
   ## 你的產業違規風險排行
   ## 最易踩的法條與罰鍰區間
   ## 對應的內控義務清單
6. 引用法條格式：法規名稱§條號（例：勞基法§59、災保法§42）
7. Cypher 使用 Neo4j v5，禁用 OVER()
```

## 常見問題排除

| 症狀 | 可能原因 | 解法 |
| --- | --- | --- |
| 畫布是空的，但 Status=Success | 瀏覽器快取 | 登出 → 重新登入 |
| 節點沒有連在一起 | Category 名稱打錯字（大小寫/空格）| 確認 Category 完全照表填寫 |
| 關係邊只連到一端 | 終點 Category 未設定 | 重新編輯 Model → Save → 清空畫布 → 全部重跑 |
| QA 回「找不到相關資料」 | Robot Setting 沒存 / PDF 還在處理中 | 確認 Documents 全部綠勾，Robot Setting 已點儲存 |
| 上傳 CSV 時欄位不對 | 打開檔案修改過格式 | 用原始 CSV，不要在 Excel 另存 |
| 數值欄位顯示文字 | 不影響功能，為原始資料格式 | 不需處理 |

## 裁罰參考資料說明（03_裁罰參考）

這些 CSV 不是 GraphRAG 節點，是給 AI 回答「同業被罰多少」時的補充資料。

| 檔案 | 用途 |
| --- | --- |
| D_全部裁罰_按法條統計.csv | 查詢哪條法規被罰最多件 |
| D_全部裁罰_按產業統計.csv | 各產業裁罰件數比較 |
| D_職安勞基_按法條統計.csv | 職安/勞基專項統計 |
| D_職安勞基_按產業統計.csv | 職安/勞基產業分布 |
| D_裁罰樣本_IT優先_2000筆.csv | 2,000 筆個案資料（IT 行業 1,500 筆 + 其他 500 筆）|

> 若平台支援，可將 `D_裁罰樣本_IT優先_2000筆.csv` 也上傳為 Document，讓 AI 可以直接引用真實案例號碼。
