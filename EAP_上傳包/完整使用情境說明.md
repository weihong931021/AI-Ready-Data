# 完整使用情境說明

> 說明使用者在 EAP 平台上提問後，系統如何透過 GraphRAG + VectorRAG 串資料，以及最終輸出什麼內容

---

## 系統運作原理（概念）

```
使用者輸入問句
      ↓
VectorRAG：從 15 份 PDF 找相關法條原文與指引段落
GraphRAG：從知識圖譜推理（工作情境→事故→法規→義務→文件→罰則）
      ↓
AI 整合兩層結果，依身分（HR/勞工）與情境（應急/止損/雷達）組合回答
```

## 情境 A：應急導航 — 職災發生後

### A-1 駐點工程師在客戶現場受傷

**你（HR）輸入：**
```
我是 IT 公司人資。工程師到客戶機房支援維修，搬伺服器時扭傷腰送醫。
客戶說現場由我們公司管，我們說是客戶指揮。這算職災嗎？誰要負責？
```

**系統在背景做的事：**

| 層 | 動作 | 結果 |
| --- | --- | --- |
| GraphRAG | 比對「駐點」「機房」「搬運」→ 命中工作情境節點 `S004` | 找到事故類型 `A003 搬運受傷` |
| GraphRAG | `A003` →（適用於）→ 法規條文 | 取出 `L001 勞基§59`、`L002 勞基§63-1 承攬連帶` |
| GraphRAG | `L001/L002` →（衍生義務）→ 企業義務 | 取出 `D002 工資補償`、`D003 醫療補償` |
| GraphRAG | `D002/D003` →（需文件）→ 文件 | 取出 `DOC001 診斷證明`、`DOC003 工資清冊` |
| GraphRAG | `A003` →（事故刑責）→ 雇主刑事責任 | 取出 `CR002 業務過失傷害` |
| VectorRAG | 搜尋「承攬連帶」「指揮從屬」→ 勞基法§63-1 原文段落 | 補充法條解釋與判斷標準 |
| VectorRAG | 搜尋「職災認定」→ 職業傷病審查準則 | 補充傷病成立要件 |

**你實際看到的輸出：**

```
## 事故類型判定
本案符合「職業災害」定義（勞基法§59）。

指揮從屬判斷：作業指示由誰發出是關鍵。
若客戶窗口指揮工程師的工作順序、搬運方式 → 客戶負主要責任
若你們公司自行安排人員、工具、流程 → 貴公司負主要責任
建議立刻蒐集：合約範圍、當日作業紀錄、門禁時間戳記

勞基法§63-1：承攬人（你的公司）與定作人（客戶）對勞工的補償責任
「連帶負補償責任」——雙方都要負，再私下求償。

---
## 法定義務（HR 接下來要做的）

| 義務 | 期限 | 說明 |
|------|------|------|
| 工資補償（勞基§59第1款）| 立即起 | 受傷期間繼續給薪，不得以傷假扣薪 |
| 醫療補償（勞基§59第1款）| 就醫即起 | 必要醫療費用由公司支付 |
| 職業傷病申報（災保法）| 就醫當下 | 協助員工向勞保局辦理職災申請 |

---
## 應備文件

- DOC001 診斷證明書（醫院開立）
- DOC003 工資清冊（計算補償基礎）
- 派工紀錄、合約書（釐清指揮關係）
- 現場照片/監視器截圖（事故發生地點）

---
## 違規罰則（若未處理）

| 違規 | 罰則 | 金額 |
|------|------|------|
| 未給付工資補償 | P002 勞基法裁罰 | 2–30 萬元 |
| 未通報職災 | P003 職安法裁罰 | 3–15 萬元 |

---
## 刑事/民事風險提示

⚠️ CR002 業務過失傷害（刑法§284）
負責人可能面臨：1 年以下有期徒刑 或 拘役 或 10 萬元以下罰金

建議：今日內完成初步文件蒐集，72 小時內與法務/HR 主管確認補償方案。
```

### A-2 通勤途中發生車禍（HR 與勞工雙面輸出）

**HR 端輸入：**
```
工程師早上去客戶端開會，途中機車被追撞，現在住院。
這算通勤職災還是因公外出？公司要做什麼？
```

**系統串資料：**
- GraphRAG：`A001 通勤事故` → `L001 勞基§59` + `L003 災保法給付`
- GraphRAG：義務鏈 → `D002~D005`；罰則鏈 → `P002/P003` → `U001 勞動局`
- VectorRAG：職業傷病審查準則「通勤路線」「合理路徑」認定標準

**HR 端看到的輸出：**
```
## 認定判斷
因公外出（比通勤保障更完整）：
- 出發點是公司/自家 → 直接前往客戶端開會 → 屬「執行職務」期間
- 審查準則：路線需合理，未繞道從事私人事務

## 法定義務（HR）
1. 協助申請災保給付（不需等認定完成，先申請）
2. 薪資補償：傷假期間工資照給
3. 醫療費墊付：公司先墊，後向勞保局申報

## 應備文件（公司準備）
- 當日出差單/行事曆（證明因公外出）
- 勞工保險災害申請書
- 診斷證明書
- 工資清冊
```

**勞工端輸入（同一事故，換身分問）：**
```
我是工程師，早上去客戶端開會途中被撞，現在住院。
我可以申請哪些補助？要準備什麼？
```

**系統串資料：**
- GraphRAG：`L001/L003` →（衍生權益）→ `B001~B005` 勞工權益節點
- GraphRAG：`B001~B005` →（受理單位）→ `U001 勞動部` / `U002 勞保局`
- VectorRAG：勞保局職災給付指南「傷病給付」「失能給付」申請流程

**勞工端看到的輸出：**
```
## 你可以申請的補助

| 給付項目 | 金額 | 申請時效 |
|----------|------|----------|
| 職災傷病給付（前2年）| 本薪 70%，住院 100% | 傷後5年內申請 |
| 醫療費用給付 | 必要醫療費全額 | 就醫後申請 |
| 失能給付（若有後遺症）| 依失能等級 1–1,200日薪 | 治療終止後申請 |
| 死亡給付（最嚴重情況）| 不適用此案 | — |

## 你要準備的文件
- 診斷證明書（醫院開立）
- 勞工保險職業災害保險傷病給付申請書（向公司HR索取或至勞保局網站下載）
- 收據正本（醫療費用）

## 申請地點
U002 勞工保險局：1955 電話諮詢，或親至各縣市勞保辦事處

## 如果公司不配合
撥打 1955 勞工諮詢專線，或向 U001 地方勞動局申訴
```

## 情境 B：精準止損 — 資遣員工前

### B-1 資遣表現不佳的工程師

**HR 輸入：**
```
要資遣一個表現持續不佳的工程師，年資 4 年，部門只有他一個人。
有哪些程序要走？期限是什麼？少做哪一步最危險？
```

**系統串資料：**

| 層 | 動作 | 結果 |
| --- | --- | --- |
| GraphRAG | `S008 資遣解僱情境` → `A006 資遣爭議` → 法規 | 取出 `L023 大量解僱§4`、`L024 勞動事件§37`、`L025 勞動事件§15` |
| GraphRAG | `L023` →（衍生義務）→ 企業義務 | `D009 解僱計畫書 60日前`、`D010 工資出勤資料` |
| GraphRAG | `L024` →（衍生權益）→ 勞工權益 | `B008 工資推定（雇主不利條款）` |
| GraphRAG | `L025` →（衍生義務）→ `D011 調解出席` | 強制調解前置 |
| GraphRAG | `D009/D010` →（需文件）→ 文件 | `DOC003 工資清冊`、`DOC010 出勤紀錄` |
| VectorRAG | 搜尋「預告期」「資遣費」→ 勞基法§16、§17 原文 | 年資 4 年預告期 30 日、資遣費計算方式 |
| VectorRAG | 搜尋「大量解僱」→ 大量解僱勞工保護法 | 確認單人不觸發大量解僱程序 |

**你實際看到的輸出：**

```
## 資遣合規檢查清單

✅ 確認資遣理由（表現不佳）
   - 需有具體績效記錄、書面警告紀錄、輔導紀錄
   - 若無書面紀錄，勞工可主張資遣「無正當理由」

✅ 是否觸發「大量解僱」？
   - 你的公司若＜30人：資遣5人以上才需依大量解僱法
   - 此案只資遣1人 → 不適用大量解僱，但仍需走預告程序

---
## 應通知事項與期限

| 步驟 | 期限 | 說明 |
|------|------|------|
| 口頭/書面預告 | 資遣日前 30 日（年資4年）| 勞基法§16，未預告須付預告工資 |
| 資遣通知書 | 同上 | 書面通知，雙方留存簽收記錄 |
| 資遣費給付 | 最後工作日 | 年資4年 = 每年0.5個月工資 × 4 = 2個月工資 |
| 非自願離職證明書 | 離職後7日內 | 供勞工申請失業給付用 |

---
## 工資爭議舉證重點

⚠️ 勞動事件法§37「工資推定」對雇主不利：
「雇主若無法提出工資資料，法院推定勞工主張的金額成立。」

你必須備齊：
- DOC003 工資清冊（完整，包含加班費明細）
- DOC010 出勤紀錄（近2年，需保存5年）
- 獎金/津貼發放依據

若上述資料不完整，日後勞工主張「積欠加班費」你無法反駁。

---
## 調解程序說明

⚠️ 勞動事件法§15「強制調解前置」：
若勞工提起訴訟，法院會先裁定「調解」，雇主必須出席。
- 無故不出席：法院可強制成立調解
- 建議：資遣前先做好工資/出勤資料，調解時才有談判籌碼

---
## 最危險的步驟（少做這個最慘）

第一名：出勤紀錄不完整
→ 勞動事件§37工資推定，整段期間的加班費都算你欠的

第二名：沒有績效/警告書面記錄
→ 勞工主張「無正當理由資遣」，可能被判回復原職
```

## 情境 C：法規風險雷達 — 主動評估公司風險

### C-1 IT 公司定期自查違規風險

**管理層/HR 輸入：**
```
我們是台北的 IT 公司，100人，主要做系統整合和軟體開發。
工程師常有專案趕工加班，但我們沒有正式的出勤打卡系統。
幫我評估今年最需要注意的勞動法規違規風險。
```

**系統串資料：**

| 層 | 動作 | 結果 |
| --- | --- | --- |
| GraphRAG | `IND_08 資訊服務` →（常見違規）→ 違規態樣 | `VP_001 工資未依規定給付`、`VP_004 工時記錄缺失`、`VP_007 未投保災保` |
| GraphRAG | `VP_001/VP_004` →（衍生法條）→ 法規條文 | `L036 §22 工資`、`L034 §24 加班費`、`L035 §30 工時記錄` |
| GraphRAG | `L034/L035` →（衍生義務）→ 企業義務 | `D006 出勤紀錄保存5年` |
| GraphRAG | `L034` →（衍生罰則）→ 罰則 | `P002`（2–100萬罰鍰）|
| GraphRAG | 裁罰案例（R8 違反法條統計）| IT行業 LBR §22：2,584件、§24：2,507件、§30：2,289件 |
| VectorRAG | 搜尋「出勤記錄」「勞動檢查」→ 勞基法施行細則 | 紀錄格式要求、保存年限規定 |

**你實際看到的輸出：**

```
## 你的產業違規風險排行（資訊服務業 IND_08）

根據 2022–2026 年 LBR 裁罰資料，資訊服務業被罰前五名：

| 排名 | 法條 | 違規內容 | 裁罰件數 | 平均罰鍰 |
|------|------|----------|----------|----------|
| 1 | 勞基法§22 | 工資未全額直接給付 | 2,584 件 | 34,200 元 |
| 2 | 勞基法§24 | 延長工時（加班費）未給付 | 2,507 件 | 29,800 元 |
| 3 | 勞基法§30 | 出勤紀錄未置備或不完整 | 2,289 件 | 22,600 元 |
| 4 | 災保法§12 | 未辦理勞工投保 | 687 件 | 20,000 元 |
| 5 | 勞基法§79 | 其他違反事項 | 412 件 | 18,500 元 |

---
## 你的公司對照分析

你說：「沒有正式的出勤打卡系統，工程師常加班」
→ 直接命中風險排行第2名（§24 加班費）和第3名（§30 出勤紀錄）

⚠️ 高風險警示：
- 勞動檢查員到公司的第一個問題：「出勤紀錄在哪裡？」
- 無紀錄 = 無法舉證加班時數 = 全部加班費算你欠的
- 加班費爭議是 IT 行業被告最多的類型

---
## 最易踩的法條與罰鍰區間

**勞基法§24 加班費未給付**
- 罰鍰：2 萬–100 萬元（按人數加重）
- 100人公司若被認定半年未給加班費：估計罰鍰 50–100 萬元起跳
- 同業真實案例：LBR_02847（台北IT公司，§24違規，罰鍰30萬）

**勞基法§30 出勤紀錄未置備**
- 罰鍰：2 萬–30 萬元
- 需保存：5 年（施行細則§21）
```

## 為什麼 Hybrid RAG 比單純 PDF 搜尋更好

| 情境 | 只用 PDF（VectorRAG）| 加上知識圖譜（GraphRAG）|
| --- | --- | --- |
| 駐點受傷 | 查到勞基§59條文，但不知道派遣連帶§63-1 | 自動展開：§59→§63-1→刑責，沒說也提示 |
| 資遣爭議 | 查到預告期規定，但漏掉工資推定陷阱 | 自動展開：L024→B008工資推定不利條款 |
| 風險雷達 | 查不到產業統計，只能查法條 | 直接從 IND_08 節點取出 IT 行業裁罰件數 |
| 核心差異 | 找到你問的，給你答案 | 找到你問的，**順便把你沒想到的也告訴你** |
